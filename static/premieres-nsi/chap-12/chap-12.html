Création de dictionnaires

Les dictionnaires constituent le seul type associatif de Python : ils permettent de réaliser une correspondance 1 à n entre valeurs de hachage (ou clés) et objets associés (valeurs).

Qu'est-ce qu'une table de hachage et quel est le rapport avec un dictionnaire ?

Les types séquences utilisent des clés numériques consécutives en tant qu'indices permettant de ranger des valeurs dans des tableaux. L'indice n'a en général rien à voir avec l'objet rangé. Il peut cependant être utile de disposer également d'une manière de ranger les objets en utilisant une valeur associée, telle qu'une chaîne de caractères, à l'objet rangé — c'est ce que nous faisons tous les jours : on enregistre des numéros de téléphones en leur associant le nom de la personne, on crée des évènements dans un calendrier en y associant leur nom, ...

Les tables de hachage sont des structures de données permettant de réaliser exactement ce qui est écrit ci-dessus. Elles permettent de ranger chaque objet, appelée valeur, en se basant sur une donnée associée, appelée clé. L'ensemble de ces deux éléments constitue une paire clé-valeur.

L'algorithme de hachage applique sur la clé une fonction de hachage qui retourne une valeur. En fonction de ce résultat l'objet est rangé à un endroit bien précis de la structure de données.

Comme l'endroit où est rangé l'objet dépend de la clé, celle-ci doit être unique dans le dictionnaire.

Il n'existe donc aucun ordre des valeurs dans une table de hachage : c'est un ensemble non ordonné de valeurs.

Les dictionnaires, en Python, sont des structures modifiables.

Comment créer un dictionnaire ?

>>> dict1 = {}  # création d'un dictionnaire vide et affectation à la variable dict1
>>> dict2 = {"nom": "mars", "type": "planète"}  # création d'un dictionnaire contenant deux élements
>>> dict1
{}
>>> dict2
{'nom': 'mars', 'type': 'planète'}

Remarque. Il est aussi possible d'utiliser la fonction intégrée dict(même si son utilisation n'est pas forcément intuitive) :

xxxxxxxxxx
>>> liste1 = ['x', 1]
>>> liste2 = ['y', 2]
>>> dict3 = dict((liste1, liste2))
>>> dict3
{'x': 1, 'y': 2}

Comment accéder aux valeurs d'un dictionnaire ?

Accès à un seul élément

La syntaxe de l'accès à un seul élément ressemble beaucoup à celle de l'accès à un élément d'une séquence, il suffit de remplacer l'indice par la clé :

xxxxxxxxxx
>>> dict2["nom"]
'mars'
>>> dict2["type"]
'planète'

Parcours d'un dictionnaire

Pour parcourir un dictionnaire, il faut parcourir les clés une à une :

xxxxxxxxxx
>>> for cle in dict2.keys():
    print("Clé : {}, valeur : {}".format(cle, dict2[cle]))
    
Clé : nom, valeur : mars
Clé : type, valeur : planète

On voit encore une fois que la boucle for en Python est une boucle Pour chaque. Il faut lire ci-dessus : « Pour chaque clé dans la liste des clés ... ».

Remarque. Pour simplifier l'accès aux éléments d'un dictionnaire, les créateurs du langage ont permis qu'il ne soit pas nécessaire d'utiliser la méthode keys pour extraire une liste de clés. Il est ainsi possible d'écrire :

xxxxxxxxxx
>>> for cle in dict2:
    print("Clé : {}, valeur : {}".format(cle, dict2[cle]))
    
Clé : nom, valeur : mars
Clé : type, valeur : planète

Lorsqu'on débute, il est tout de même plus clair d'utiliser la méthode keys, c'est donc ce que vous ferez.

Comment modifier un dictionnaire ?

On peut mettre à jour un dictionnaire en lui ajoutant une nouvelle entrée, en modifiant une valeur ou en supprimant une valeur.

Modification

xxxxxxxxxx
>>> dict2['nom'] = "soleil"
>>> dict2["type"] = "étoile"
>>> dict2
{'nom': 'soleil', 'type': 'étoile'}

Ajout

xxxxxxxxxx
>>> dict2["masse"] = 1.989E30
>>> dict2["rayon"] = 695510000
>>> dict2
{'nom': 'soleil', 'type': 'étoile', 'masse': 1.989e+30, 'rayon': 695510000}

Destruction

xxxxxxxxxx
>>> del dict2["nom"]  # supprime la valeur de clé "nom"
>>> dict2.clear()     # supprime toutes les valeurs de dict2 (le dictionnaire existe toujours)
>>> del dict2         # supprime le dictionnaire en entier

Quelques opérateurs et méthodes à connaître

La clé existe-t-elle dans le dictionnaire ?

xxxxxxxxxx
>>> "rayon" in dict2.keys()  # "rayon" appartient-elle à la liste des clés ?
True
>>> "rayon" in dict2         # version plus concise
True
xxxxxxxxxx
>>> "masse" not in dict2.keys()  # "masse" n'apartient pas à la liste de clés ?
False
>>> "masse" not in dict2         # idem
False
>>> "température" not in dict2.keys()
True

Combien d'éléments le dictionnaire renferme-t-il ?

xxxxxxxxxx
>>> len(dict2)           # le dictionnaire comme argument à la fonction len
4
>>> len(dict2.keys())    # la liste des clés comme argument à la fonction len
4
>>> len(dict2.values())  # la liste des valeurs comme argument à la fonction len
4

Listes des valeurs et des clés

xxxxxxxxxx
>>> dict2
{'nom': 'soleil', 'type': 'étoile', 'masse': 1.989e+30, 'rayon': 695510000}
>>> dict2.keys()
dict_keys(['nom', 'type', 'masse', 'rayon'])
>>> dict2.values()
dict_values(['soleil', 'étoile', 1.989e+30, 695510000])

Remarque. Les méthodes values et keys ne retournent en fait pas des listes mais des itérateurs. La différence n'est pas importante tant que vous vous utilisez le résultat de leur appel dans des instructions comme for ou in :

xxxxxxxxxx
>>> 'étoile' in dict2.values()
True
>>> for valeur in dict2.values():
    print(valeur, end=" ")
    
soleil étoile 1.989e+30 695510000
